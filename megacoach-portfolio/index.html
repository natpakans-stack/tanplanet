อยากดู dashboard บน cloud แต่ห้าม cloud เห็นเงิน

deploy static ขึ้น Vercel = ใครเปิดลิงก์ก็เห็น ถ้าฝังตัวเลขจริงลงไฟล์ = มูลค่าพอร์ตหลุดทันที

แก้: เข้ารหัส payload ด้วย AES-256-GCM + PBKDF2 250k ในเครื่องก่อน แล้ว push เฉพาะ ciphertext · dashboard ถอดรหัสฝั่ง client ด้วย Web Crypto หลังผู้ใช้ใส่ passphrase — cloud ถือแต่ blob ที่อ่านไม่ออก
🔒

LINE webhook verify signature ไม่ผ่าน

ต้อง verify x-line-signature จาก raw body — แต่ Vercel parse body เป็น object ให้ก่อน ทำให้ HMAC ที่คำนวณไม่ตรง เลย reject ของจริง

แก้: ปิด parser (bodyParser: false) แล้วอ่าน raw stream เอง → คำนวณ HMAC-SHA256 บน byte ดิบเป๊ะ ๆ ก่อนค่อย JSON.parse — signature จึงตรง กันคนยิง webhook ปลอมได้จริง
🖥️

เลขต้องสดทุกวัน แต่คีย์ห้ามออกจากเครื่อง

ถ้าให้ cloud คำนวณมูลค่า = ต้องเอาคีย์ขึ้น cloud (ขัดกับ zero-knowledge) แต่ถ้าไม่อัปเดต ตัวเลขก็ค้าง

แก้: ให้ Mac เป็น source of truth — launchd ปลุก Bun script ทุกเช้า ดึงราคา → คำนวณ → เข้ารหัส → git push เอง · คีย์อ่านจาก macOS Keychain ไม่เคยขึ้น cloud · git push กลายเป็น deploy pipeline
📈

cloud routine ต้องรู้ "โซนราคา" แต่ห้ามรู้ "จำนวนหุ้น/เงิน"

routine ฝั่ง cloud ต้องเทียบราคาสดกับ entry/stop เพื่อเตือน — แต่ข้อมูลพวกนั้นอยู่ปนกับจำนวนหุ้นและมูลค่าจริง

แก้: สร้าง publish-levels ที่คัดออกมาเฉพาะ ticker + โซน + held-flag ลง JSON สาธารณะ · มี --check รัน regex กันคำอย่าง share/cash/value/บาท หลุด — ผิดเมื่อไหร่ build fail
🔔

เตือนซ้ำ ๆ ทุกวันจนน่ารำคาญ

ราคายังอยู่ใต้ stop หลายวันติด = ถ้ายิงทุกวันคนก็เลิกอ่าน · แถม Mac กับ cloud อาจเตือนเรื่องเดียวกันซ้อนกัน

แก้: cooldown ราย ticker:kind (เตือนแล้วเงียบ N วัน) + แบ่งหน้าที่ชัด — Mac ยิงเฉพาะ stop/trim ที่ safety-critical, cloud รับ entry nudge ไป กันเตือนซ้ำและกัน alert fatigue
☕

ดึงราคาพลาดตอนเครื่องเพิ่งตื่นจาก sleep

launchd ปลุกงานตอน Mac เพิ่งตื่น — บางครั้ง WiFi ยังไม่กลับมา ทำให้ fetch ราคาแรก ๆ ล้มเหลว แล้วงานทั้งรอบพัง

แก้: ห่อ fetch ด้วย retry 3 ครั้ง + หน่วง 4 วินาทีต่อรอบ (เผื่อเน็ตกลับ) · ทั้ง flow ออกแบบ best-effort — pull ไม่ได้ก็ update ต่อ, publish พลาดก็ไม่ล้มทั้งงาน — บทเรียน: งาน cron ต้องทน network ไม่พร้อม

Level Up · Reflect

Knowledge Domain ที่แข็งขึ้น

5 พื้นที่ความรู้ที่ระดับเราขยับขึ้นจริงจากการทำโปรเจกต์นี้

1. Applied Cryptography — จาก "รู้ชื่อ" เป็น "ออกแบบระบบ zero-knowledge ได้":

ลงมือใช้ Web Crypto จริง — PBKDF2 derive key จาก passphrase, AES-256-GCM encrypt/decrypt, จัดการ salt/iv/base64 เอง แล้วออกแบบ flow ให้คีย์ไม่เคยออกจากเครื่อง (relay รับแต่ ciphertext) บทเรียน: zero-knowledge ไม่ใช่ magic — มันคือการวางขอบเขตว่า "ใครถือคีย์" ให้ถูกที่

2. Serverless + Chat-platform engineering — LINE/LIFF + webhook เข้าใจทะลุ:

ต่อ LINE Messaging API (reply/broadcast), ฝัง LIFF webview, verify webhook signature บน raw body, ออกแบบ Flex bubble ที่ปรับสีตาม state — และเข้าใจ quirk ของ serverless (body parsing, cold start, stateless) บทเรียน: การ integrate platform จริงคือการอ่าน spec + จัดการ edge case ของ runtime

3. Automation & source-of-truth design — cron ที่ทนโลกจริง:

วางสถาปัตยกรรม "เครื่องเดียวถือความจริง" — launchd orchestrate หลาย Bun script ต่อกัน (pull → update → publish → seed), ใช้ git push เป็น deploy, ออกแบบทุกขั้นให้ best-effort (network พร้อมไม่พร้อมก็ไม่ล้มทั้งงาน) บทเรียน: ระบบอัตโนมัติที่ดีต้องคิดเผื่อวันที่ทุกอย่างไม่พร้อม

4. LLM product engineering — จาก "เรียก API" เป็น "ออกแบบพฤติกรรมโค้ช":

เขียน system prompt ที่ฝังปรัชญาการลงทุน (Fear of Ruin, core vs satellite), inject context พอร์ตแบบ dynamic ก่อนถาม, เปิด tool web_search ให้ดึงข้อมูลสด, และ post-process คำตอบ (strip citation/artifact) บทเรียน: คุณภาพผู้ช่วย AI อยู่ที่การ "กำกับบริบทและขอบเขต" มากกว่าตัวโมเดล

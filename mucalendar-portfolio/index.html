ศาสตร์โบราณถูกจำกัดอยู่กับหมอดู

Design System ที่ต้องสร้างจาก 0

ทีมไม่มี Design System มาก่อน ต้องกำหนด Tokens ตั้งแต่ สี Typography Spacing Radius ทุกอย่าง — ใช้เวลา Week แรกทั้ง Week แค่วาง Foundation

สร้าง Design Tokens เป็นระบบ ทำให้ Week 2-3 ทำงานเร็วขึ้น 3 เท่า
✨

Animation ไพ่ทาโรต์ที่ต้อง "รู้สึก" ได้

ไพ่ทาโรต์ไม่ใช่แค่เปิดปิด — ต้องมี 3D flip, shimmer, glow effect ให้รู้สึกถึง "ความศักดิ์สิทธิ์" ทดลอง CSS transform 3D หลายรอบจนได้ feel ที่ใช่

ใช้ CSS perspective + preserve-3d + cubic-bezier ทำ 3D flip ที่ smooth โดยไม่พึ่ง JS library
🤝

3 คน 3 สไตล์ — จะ Merge ยังไง?

แต่ละคนออกแบบคนละ flow — สไตล์ Spacing Color ต่างกันหมด ถ้าไม่มีกติกา output จะไม่เป็นเนื้อเดียวกัน

แก้ด้วย Design Token Sheet + Component Library ที่ทุกคนใช้ร่วมกัน — Review กันทุก Week
📐

Responsive ที่ต้องดูดีทุก Breakpoint

แอปดูดวงมี Content เยอะมาก — Card Fortune, ไพ่ 22 ใบ, Dashboard ต้อง Layout ให้ดีทั้ง Mobile/Tablet/Desktop โดยไม่หาย

ใช้ CSS Grid + clamp() + Container Query คิดแบบ Mobile First แล้วขยายขึ้น

Key Takeaways

สิ่งที่ทีมได้เรียนรู้จากการทำ Daily MU

Design System ช่วยได้จริง — แต่ต้องลงทุนก่อน:

Week แรกรู้สึกช้ามาก เพราะเอาเวลาไปสร้าง Tokens/Components แต่พอ Week 2-3 ทุกคนทำงานเร็วขึ้นมาก เพราะไม่ต้องตัดสินใจเรื่องสี/Spacing ใหม่ทุกครั้ง

Collaboration = Communication:

3 คนทำงานพร้อมกัน ถ้าไม่คุยกัน output จะไม่เป็นเนื้อเดียว — Review ทุก Week + ใช้ Design Token ร่วมกัน ทำให้ Merge งานได้ smooth

Animation ต้อง "รู้สึก" ไม่ใช่แค่ "เคลื่อนไหว":

ไพ่ทาโรต์ เซียมซี ต้องให้ผู้ใช้รู้สึกถึง "ความพิเศษ" — ไม่ใช่แค่ transition ธรรมดา แต่ต้องมี timing, easing, glow ที่สร้าง emotional moment

Mobile First ช่วยป้องกันปัญหา Responsive:

ออกแบบ Mobile ก่อนแล้วขยาย — ทำให้ Content ไม่หายเวลาย่อจอ และ Layout ไม่พังเมื่อเจอข้อมูลจริง
